先解决采集这一步
Node_ID: exporter

这里简单说一下它背后的实现。

文章采集这一块，我用到的是一个开源项目，叫 wechat-article-exporter。[1]

它本身是一个免费的微信公众号文章导出工具，可以帮助我们搜索公众号文章，并把文章内容导出来。

这里解决是怎么采集数据，为了给后面的分析做好准备。整体使用流程大概是这样：

   terminalTEXT
1. 首先登录平台
2. 然后获取对应的 `API` 密钥
3. 把密钥交给 HermesAgent
4. 让 Agent 按照你配置的信息源去获取文章
5. 后面再做分析、推送和存储

先看登录这块。

登录平台

这里需要注意一下。我们在登录的时候，需要使用微信公众号相关能力。

如果你自己还没有公众号，可以先去注册一个。

因为它用到的原理大概是：公众号后台在写文章的时候，本身支持搜索其他公众号的文章。

这个项目就是利用这个能力，把公众号文章检索和导出这一步做了出来。

登录之后，再去获取 API 密钥。

获取API秘钥

拿到密钥之后，后面就可以准备着把它交给 Hermes Agent 使用。

  
封装成了一个 Skill
Node_ID: skill

不过说实话，让大家自己从 GitHub 项目开始看，再去研究怎么部署、怎么配置、怎么接到 Agent 里，还是有一点门槛的。

所以我直接把这套流程整理成能复用的Skill，方便大家一键安装

你只需要提供要监听的公众号，还有刚刚说到的 API 密钥发给自己的 Hermes Agent，或者小龙虾🦞，让它帮你安装和配置就可以了。

先看下这个 Skill 里大概有什么：

   wechat-mp-monitorTEXT
wechat-mp-monitor/
├── SKILL.md# 使用说明文档
├── scripts/
│   └── mp-monitor.py#公众号监控脚本
├── state/
│   └── mp-monitor.json# 状态文件，记录已采集内容
└── cache/
└── mpdashboard.html# Dashboard页面

简单说，每一部分的作用是：

SKILL.md：告诉 Agent 这个 Skill 怎么触发怎么用
mp-monitor.py：负责执行公众号文章监控
mp-monitor.json：记录已经采集过的文章，避免重复
mp-dashboard.html：生成一个简单的工作台页面，方便查看文章列表

配置好之后，就可以直接交给 Hermes Agent 安装。

交给Hermes安装

安装好了，可以飞书里问下测下，如果看到出现卡片就算成功了。到这儿整个流程就算跑通了，后面配上定时任务，就ok了。
